## 当前状态

---

## 目录结构

```
workspace/
├── inbox.md          # 收集箱：新想法、待办事项先扔这里
├── next-actions.md   # 下一步行动：已拆解可执行的任务
├── projects.md       # 项目清单：多步骤大目标
├── waiting.md        # 等待中：需他人回复或条件成熟的事项
├── delegated.md      # 已委派：交给子任务的事项
├── someday.md        # 将来也许：不急但不想忘的灵感
└── done/             # 按周归档已完成的任务
```

---

## 核心原则

1. **收集 → 整理 → 执行 → 归档**
2. 任务流动：inbox → next-actions → done
3. 每周日周报中包含 GTD 回顾

---

## 应用场景

**inbox** → Holly 说的新想法、需要确认的事项
**next-actions** → 已确认可执行的任务、定时任务
**projects** → 课题、讲座等长期目标
**waiting** → 等 Holly 确认
**someday** → 以后想做但现在不急的事
