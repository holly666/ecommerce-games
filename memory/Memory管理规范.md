# Memory管理规范

## 原则：精而不滥，定期清理

---

## 索引文件（不变）

**MEMORY.md** - 轻量索引
- 只有主题文件名和更新时间
- 不存具体内容

---

## 核心档案

| 文件 | 内容 | 更新频率 |
|------|------|----------|
| USER.md | Holly基础信息（年龄、职业、家庭、偏好、忌讳） | 有变化就改 |
| 备孕.md | 调理方案、周期、就诊记录 | 有变化就改 |
| 投资.md | 持仓、策略 | 有变化就改 |
| 教学.md | 课程安排、课题进度 | 有变化就改 |
| 健康.md | 饮食、运动习惯 | 有变化就改 |

**关于USER.md**：
- 只存Holly本人的基础信息
- 具体专题内容归各专题文件
- 偏好/忌讳可以存在这里，也可以引用专题文件

**关于其他人**：
- 需要时单独建文件（如 家人.md、学生某某某.md）

---

## 每日记录（定期归档）

**memory/YYYY-MM-DD.md** - 当天会话流水
- 每天自动生成
- 归档：超过30天压缩到archive/

---

## 进化记录

**memory/每日进化.md** - 纠错+进化
- 每次被她纠正后更新
- 每月review一次

---

## 规则

1. **不重复**：Obsidian有的，这里不存第二份
2. **不流水**：不记聊天内容，只记结论
3. **定期清**：每日复盘时检查，有用的留下，没用的删
4. **要引用**：提到什么，注明来自哪个文件
5. **场景化读取**：
   - 推荐时 → 读偏好层
   - 闲聊时 → 读状态层
   - 深度决策时 → 读人格层

---

## 月度Review规则

**沉淀条件**：

| 状态层信息 | 沉淀到 | 条件 |
|-----------|--------|------|
| 连续1个月没变化的习惯 | 人格层 | 形成稳定习惯了 |
| 反复出现的偏好 | 偏好层 | 聊到3次以上 |
| 临时状态 | 删除 | 已结束（如感冒、临时任务） |
| 阶段性目标 | 保留+标注 | 还在进行中 |

**例子**：
- "凌晨3-4点起来上厕所" → 连续1个月都在说 → 沉淀到偏好层
- "最近在调理" → 调理结束 → 移入历史
- "血糖偏高" → 一直持续 → 保留在状态层
