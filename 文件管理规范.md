# 文件管理规范

> 更新：2026-03-11

---

## 一、版本号规则

### 格式
```
V{主版本}.{次版本}-{日期}
```

### 版本定义

| 版本 | 适用场景 |
|------|----------|
| V1.0 | 初版/重写版 |
| V1.x | 小幅修订（润色、优化、修改） |
| V2.0 | 大幅修改（结构调整、内容重写） |
| V3.0 | 再次修改 |
| FINAL | 最终定稿 |

### 示例
```
结题报告-智链乡村-V1.0-20260311.md   ← 初版
结题报告-智链乡村-V1.1-20260311.md   ← 小幅修订
结题报告-智链乡村-V2.0-20260311.md   ← 大幅修改优化
结题报告-智链乡村-FINAL-20260311.md  ← 最终定稿
```

---

## 二、文件命名

### 课题类
```
{类型}-{课题名}/内容描述}-V{版本号}-{日期}.扩展名
```

### 日常类
```
[日期]-[类型]-[描述].扩展名
```

常用前缀：笔记、清单、模板、参考、汇总、纪要、汇报

---

## 三、核心原则

1. **一版一文件** — 每次修改新建文件，不要覆盖
2. **版本号递增** — 不能重复版本号
3. **日期统一** — 用 YYYYMMDD 或 YYMMDD 格式
4. **英文优先** — 避免飞书/微信传输乱码

---

## 四、操作流程

### 修改文件的正确姿势
1. 复制原文件
2. 改版本号（V1.0 → V1.1）
3. 在新文件中修改
4. 保存新文件

### 禁止操作
- ❌ 直接覆盖原文件
- ❌ 重复使用相同版本号
- ❌ 不带版本号保存
