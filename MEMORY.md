## 硬规则（纠错达3次自动升级）

- **未经确认不得执行** — 外部操作需先确认
- **路径不确定时用搜索** — 用户提到文件/库名，不要猜路径，直接 `mdfind` 或 `find` 搜
- **重要信息必须存档** — 用户给的重要链接/资料，必须存到Obsidian，不能只记在对话上下文里
- **喜好及时记录** — 关于Holly的喜好、习惯、忌讳要第一时间记录到 USER.md
- **需求不明确时先确认** — 不确定用户要什么，先问清楚再动手
- **记忆调用要准确** — 记录过的信息需要用时必须调取出来，不能忘了去查
- **技能名必须Exact match** — 用户说下载"xxx技能"，必须Exact match名称，不一样名字的不要下载
- **看到图片就触发OCR** — 用户发截图/图片，第一反应是用PaddleOCR识别，而不是说"没法识别"
- **PPT默认用pptx技能** — 做PPT优先用 anthropics/skills@pptx，不是自己写代码
- **搜索默认用tavily技能** — 通用搜索优先用 tavily_search，金融搜索用东方财富技能
- **风险等级不是HIGH可自行下载** — 安全审查通过后，低风险和中风险技能可以直接下载，无需再问用户
- **承诺的事必须闭环** — 答应设置提醒/记录等，必须立即执行+确认完成，不能只是"说要做"
- **网页互动游戏流程** — 基于 /Users/holly/project/ecommerce-games 项目 → 写完提交git → 给GitHub Pages网址
- **发文件给用户** — 先复制到 ~/.openclaw/workspace/ 再用filePath发送
- **定时任务用OpenClaw Cron** — 设置定时提醒/任务，用 openclaw cron add，不用系统crontab
- **饮食建议先读清单** — 给 Holly 饮食建议前，必须先读取 ~/Documents/Holly工作生活笔记/健康/孕期饮食禁忌清单

## Cron vs Heartbeat 使用规则

| 场景 | 用 Cron | 用 Heartbeat |
|------|---------|--------------|
| 固定时间提醒（每天/每周几点） | ✅ 每天15:00、每周一9:00 | ❌ |
| 需要精确时间 | ✅ | ❌ |
| 需要获取实时动态信息（天气、课程、股票） | ❌ | ✅ 早间简报、孕期阶段 |
| 周期性检查（邮箱、消息） | ❌ | ✅ |
