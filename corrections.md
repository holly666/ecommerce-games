### 2026-03-24 问题：GitHub仓库误提交大量无关文件

**问题描述**
- 在workspace目录直接执行git操作，没有先检查是否已有git仓库

**原因分析**
- 没有先检查`git status`确认加了哪些文件
- 应该在临时目录准备好干净的文件再push

### 2026-03-29 问题：奇数天判断基准搞错

**问题描述**
- 连续3次在黄体酮针/阿司匹林服药规则上犯错
- 根本原因：以为"奇数天"是按日历日期单双（29号=奇数），实际是从治疗起始日（03-27）开始数天数

**正确规则**
- 治疗起始日：03-27（黄体酮开始日）
- 天数计算：(今天 - 起始日 + 1) = 第N天
- 第1/3/5/7...天 = 奇数天
- 验证：03-27=第1天(奇)不打，03-28=第2天(偶)打，03-29=第3天(奇)不打 ✅

**教训**
- 遇到"从X开始每奇数天..."这种表述，必须先确认起始日
- 不能套用日历单双号，必须代入起始日验算
- 以后配置写完后必须用实际日期代入验算一次

### 2026-03-29 问题：memoryId操作必须加scope参数

**问题描述**
- 用 `memory_delete(memoryId=xxx)` 删除记忆时报 "outside accessible scopes"
- 用 `memory_update(memoryId=xxx)` 更新也报同样错误
- 但用 `memory_delete(query="xxx")` 搜索删除就能成功

**根本原因**
- memoryId 操作需要同时传 `scope=agent:main` 参数
- query 搜索不需要 scope，因为 query 是跨 scope 搜索

**正确用法**
```python
# ✅ 正确：memoryId + scope
memory_delete(memoryId="abc123", scope="agent:main")

# ✅ 正确：query 搜索不需要 scope
memory_delete(query="关键词")

# ❌ 错误：只有 memoryId 没有 scope
memory_delete(memoryId="abc123")  # 报错
```

**其他发现**
- lancedb-pro 没有物理删除的"遗忘"机制
- 新版会覆盖旧版优先级，旧版渐渐查不到
- update 失败时：store 新版本 + query 删除旧版本

**教训**
- 删/改记忆时，memoryId 搭配 scope 参数一起用
- 确认失败时换方案（store+delete）而不是死磕

**状态**：✅ 已修复并沉淀
